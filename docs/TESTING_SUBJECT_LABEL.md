# Subject Label Feature Testing Guide

## 테스트 환경 설정

### 1. Extension 로드
```bash
npm run build:local
```
- Chrome에서 `chrome://extensions/` 접속
- "개발자 모드" 활성화
- "압축 해제된 확장 프로그램을 로드합니다" 클릭
- `/dist` 폴더 선택

### 2. eCampus 로그인 (선택사항)
- eCampus에 로그인하여 수강 과목 목록 가져오기
- 로그인하지 않아도 라벨 시스템은 정상 작동

## 테스트 시나리오

### 시나리오 1: 기본 과목 라벨 생성
**목표**: 새로운 과목 라벨을 생성하고 색상을 지정할 수 있다

1. ✅ LinKU 확장 프로그램 열기
2. ✅ "Add Todo" 버튼 클릭
3. ✅ "과목명" 필드 클릭
4. ✅ "웹프로그래밍" 입력
5. ✅ 드롭다운에서 "'웹프로그래밍' 라벨로 추가" 클릭
6. ✅ 색상 팔레트에서 건대 녹색(#007a30) 선택
7. ✅ "생성" 버튼 클릭
8. ✅ 입력 필드에 "웹프로그래밍"이 유지되는지 확인

**예상 결과**: 
- 라벨이 생성되고 색상이 저장됨
- 드롭다운이 자동으로 닫힘

### 시나리오 2: 생성된 라벨 선택
**목표**: 이전에 생성한 라벨을 선택할 수 있다

1. ✅ "과목명" 필드 다시 클릭
2. ✅ 드롭다운에서 "웹프로그래밍" 항목 확인
3. ✅ 녹색 점(dot)이 과목명 옆에 표시되는지 확인
4. ✅ 체크 아이콘이 우측에 표시되는지 확인
5. ✅ "웹프로그래밍" 클릭

**예상 결과**:
- 라벨이 선택되고 입력 필드에 표시됨
- 사용 횟수가 증가함

### 시나리오 3: eCampus 과목 제안 (eCampus 로그인 필요)
**목표**: eCampus에서 가져온 과목을 제안받을 수 있다

1. ✅ eCampus에 로그인
2. ✅ LinKU 새로고침
3. ✅ "Add Todo" 버튼 클릭
4. ✅ "과목명" 필드 클릭
5. ✅ eCampus 과목들이 드롭다운에 표시되는지 확인
6. ✅ eCampus 과목은 색상 점이 없음 확인
7. ✅ eCampus 과목 하나 선택

**예상 결과**:
- eCampus 과목이 제안 목록에 표시됨
- 선택 가능

### 시나리오 4: 검색 및 필터링
**목표**: 입력하는 동안 제안이 필터링된다

1. ✅ "과목명" 필드에 "프로" 입력
2. ✅ "프로"가 포함된 과목만 표시되는지 확인
3. ✅ "데이터베이스" 입력
4. ✅ 일치하는 항목이 없으면 빈 상태 메시지 확인

**예상 결과**:
- 실시간 필터링 작동
- 일치하는 항목만 표시

### 시나리오 5: Todo 생성 및 색상 표시
**목표**: 생성된 Todo에 과목 라벨 색상이 표시된다

1. ✅ 할 일: "과제 제출" 입력
2. ✅ 과목명: 이전에 만든 "웹프로그래밍" 선택
3. ✅ 마감일/시간 입력
4. ✅ "추가" 버튼 클릭
5. ✅ Todo 리스트에서 생성된 항목 확인
6. ✅ "웹프로그래밍" 옆에 녹색 점이 표시되는지 확인

**예상 결과**:
- Todo가 생성됨
- 과목명 옆에 라벨 색상이 원형으로 표시됨

### 시나리오 6: 여러 과목 라벨 생성 및 정렬
**목표**: 여러 라벨을 생성하고 사용 빈도 기반 정렬을 확인한다

1. ✅ "자료구조" 라벨 생성 (빨강색)
2. ✅ "알고리즘" 라벨 생성 (파랑색)
3. ✅ "자료구조"로 Todo 2개 생성
4. ✅ "알고리즘"으로 Todo 1개 생성
5. ✅ "웹프로그래밍"으로 Todo 1개 생성
6. ✅ "과목명" 필드 클릭하여 드롭다운 열기
7. ✅ "자료구조"가 최상단에 표시되는지 확인 (가장 많이 사용됨)

**예상 결과**:
- 사용 빈도가 높은 라벨이 상단에 표시됨
- 각 Todo에 올바른 색상이 표시됨

### 시나리오 7: 드롭다운 외부 클릭
**목표**: 드롭다운 외부를 클릭하면 닫힌다

1. ✅ "과목명" 필드 클릭하여 드롭다운 열기
2. ✅ 다른 입력 필드 클릭
3. ✅ 드롭다운이 닫히는지 확인
4. ✅ 색상 선택 모드일 때도 외부 클릭 시 닫히는지 확인

**예상 결과**:
- 외부 클릭 시 드롭다운이 닫힘

### 시나리오 8: 라벨 없이 Todo 생성
**목표**: 과목명 없이도 Todo를 생성할 수 있다

1. ✅ "Add Todo" 버튼 클릭
2. ✅ 할 일만 입력하고 과목명은 비워둠
3. ✅ "추가" 버튼 클릭
4. ✅ Todo가 정상적으로 생성되는지 확인

**예상 결과**:
- 과목명 없이도 Todo 생성 가능
- 과목명 필드가 선택사항임을 확인

## Edge Cases

### Edge Case 1: 중복 라벨 생성 방지
1. ✅ "웹프로그래밍" 라벨이 이미 있는 상태
2. ✅ 다시 "웹프로그래밍" 입력하고 라벨로 추가 시도
3. ✅ 기존 라벨이 반환되어야 함 (중복 생성 안 됨)

### Edge Case 2: 빈 문자열 라벨 생성 방지
1. ✅ "과목명" 필드를 비워둔 상태에서 라벨 추가 버튼 클릭
2. ✅ 아무 동작도 하지 않아야 함

### Edge Case 3: 매우 긴 과목명
1. ✅ 50자 이상의 긴 과목명 입력
2. ✅ UI가 깨지지 않고 적절히 표시되는지 확인

- ✅ Chrome (주 타겟)
- ⚠️ Edge (Chromium 기반, 동일하게 작동 예상)
- ❌ Firefox, Safari (Chrome Extension 전용)

## Known Issues / Limitations

1. 라벨 편집 기능 없음 (향후 추가 예정)
2. 라벨 삭제 기능 없음 (향후 추가 예정)
3. 커스텀 색상 입력 불가 (8가지 팔레트만 지원)
4. 라벨 순서 수동 조정 불가 (사용 빈도 기반만 지원)
